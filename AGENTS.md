# AGENTS.md

## Общие правила

-   Работать строго по этим правилам.
-   Ничего не додумывать и не делать "по своему" вне рамок задачи.
-   Цель изменений: лучшее решение по
    качеству/надёжности/поддерживаемости в рамках задачи.
-   Приоритет: корректность, безопасность, читаемость, поддерживаемость.
-   Улучшения допускаются только если они напрямую усиливают решение
    задачи.

------------------------------------------------------------------------

## 1. Сначала документация (MCP) --- потом код

Перед изменениями кода обязательно:

1)  Проверить context7\
2)  Сравнить подходы\
3)  Предложить 2--3 варианта\
4)  Дождаться явного выбора пользователя

Без выбора реализация запрещена.

------------------------------------------------------------------------

## 2. Работа только через ветки и новый PR

Запрещено: - Работать в main/master - Делать merge - Допушивать в
существующий PR

Каждое изменение: 1. Новая ветка 2. Commit 3. Push 4. Новый Pull Request

Merge делает только владелец.

------------------------------------------------------------------------

## 3. План перед изменениями

Перед реализацией: - Описать план - Указать файлы - Зафиксировать
Definition of Done

Без DoD реализация запрещена.

------------------------------------------------------------------------

## 4. Показ изменений

Перед PR показать: - git diff --stat - ключевые части diff

------------------------------------------------------------------------

## 5. Зависимости

Нельзя менять зависимости без разрешения.

------------------------------------------------------------------------

## 6. Тесты

Если есть тесты --- запускать и показывать результат.

------------------------------------------------------------------------

## 7. Безопасность

-   Не коммитить секреты
-   Использовать .env.example

------------------------------------------------------------------------

## 8. Коммиты

-   fix:
-   feat:
-   refactor:
-   chore:

------------------------------------------------------------------------

## 9. Язык общения

Все объяснения --- на русском языке.

------------------------------------------------------------------------

## Handoff

После завершения задачи агент обязан:

1.  Создать НОВЫЙ Pull Request.
2.  Остановиться.

### Обязательная проверка Docker

После любой правки кода:

1.  docker compose build\
2.  docker compose up -d\
3.  Проверить работоспособность сайта

Если проект не запускается:

-   Найти причину\
-   Исправить\
-   Повторно пересобрать\
-   Добиться корректного запуска\
-   Только после этого создавать Pull Request

PR создаётся только при подтверждённой работоспособности в Docker.
